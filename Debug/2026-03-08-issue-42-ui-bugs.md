# 2026-03-08 issue 42 UI bug 修复记录

## 问题

- 仪表盘最近活动卡片中的实时日志流存在“无法展开”的情况。
- 智能体页面“转为显式配置”弹窗会被顶部标题栏遮挡。
- 技能中心等页面右侧滚动条出现时，右侧按钮区域会有遮挡和闪烁感。

## 定位

- `web/src/pages/Dashboard.tsx` 仅在 `detail` 存在时才允许展开，但很多日志只有 `summary`，导致用户看到可点击列表却没有可读的展开内容。
- `web/src/pages/Agents.tsx` 的弹窗层级是 `z-50`，低于现代布局页头的高层级元素。
- `web/src/index.css` 的现代滚动容器没有预留稳定滚动条槽位，出现滚动条时内容宽度会抖动。

## 修复

- 仪表盘日志展开改为优先展示 `detail`，没有 `detail` 时回退展示 `summary`。
- 智能体编辑弹窗层级提升到高于主布局头部，避免遮挡。
- 为 `ui-modern-scrollbar` 增加 `scrollbar-gutter: stable`，减少按钮区因滚动条出现造成的抖动与遮挡。

## 验证建议

- 在仪表盘点击最近活动中的普通日志和详细日志，确认都能展开内容。
- 在智能体页面打开“转为显式配置”弹窗，确认顶部不会被导航标题栏压住。
- 在技能中心和其他长列表页面滚动，确认右侧按钮不再因滚动条出现而闪烁位移。
