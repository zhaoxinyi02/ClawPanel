# 已知问题与待完善项

这里整理当前仓库中值得持续跟进的兼容性问题、安装体验问题和待完善项。

## Windows PowerShell 5.1 兼容性

### 现象

- 在旧版 Windows PowerShell 5.1 环境中运行安装脚本时，可能因为新语法不兼容而报错。

### 建议

- 优先使用 PowerShell 7+
- 如果是历史脚本导致的问题，请先更新安装脚本后再重试

## 安装脚本版本获取方式

### 现状

- 安装脚本、更新脚本与发布元数据之间需要保持严格同步
- 如果元数据或加速源同步延迟，用户可能拿到旧版本信息

### 建议方向

- 持续收敛到统一的发布元数据源
- 避免脚本中存在多处手动维护的版本号入口

## Windows 一键安装 OpenClaw 的依赖检查体验

### 现状

- Windows 环境下一键安装 OpenClaw 时，若依赖检查过于严格，可能在缺少 Node.js 时直接退出

### 建议方向

- 提供更明确的依赖提示
- 优先给出自动安装或补救路径，而不是直接失败退出

## 使用建议

- 安装 / 部署类问题优先在 Discussions 中交流
- 可以稳定复现的缺陷优先提交 Issue
- 历史兼容性问题与典型排障案例会逐步从零散备忘整理到 `docs/` 和 Wiki
