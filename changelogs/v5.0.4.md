# ClawPanel v5.0.4 更新日志

发布日期：2026-02-27

## 新功能

### NapCat 一键诊断修复
- 新增「检测状态」和「诊断并修复」按钮，位于通道管理 QQ 页面
- 8 步全面检测：Docker 安装 → 服务状态 → 容器运行 → 端口映射 → OneBot11 配置 → WebUI 配置 → 端口可达性 → QQ 登录状态
- 自动修复模式：启动容器、写入默认配置、设置 WebUI Token、重启容器
- 支持 Windows Shell 模式和 Linux Docker 模式

### 系统诊断报告
- 系统配置 → 运行环境新增「生成诊断报告」按钮
- 输出完整环境信息：操作系统、软件版本、OpenClaw 配置、模型提供商、通道、技能、NapCat 状态、PATH 等
- 报告可一键复制，方便反馈问题

## 优化

### NapCat 使用体验优化
- 扫码登录后自动轮询检测登录状态（每 3 秒），登录成功后自动关闭弹窗
- 重启 NapCat 时显示 Loading 转圈动画和提示
- 二维码刷新后自动重新启动轮询

### Docker 入口脚本改进
- 禁用自动登录上次 QQ 账号，每次启动均需手动登录（QR/快速/密码）
- 解决手机端退出后 NapCat 仍尝试旧会话的问题
- 用户仍可通过面板快速登录功能复用历史会话

### NapCat 安装脚本修复
- 安装完成后自动重启容器使配置生效
- 添加 WebUI (6099)、WS (3001)、HTTP (3000) 端口就绪检测
- 端口未就绪时给出明确提示

## 修复

- 修复 OpenClaw 环境检测 PATH 不完整问题，补充 `~/.local/bin` 和 `~/.npm-global/bin`
- 修复因 PATH 缺失导致技能数量检测为 0 的问题
- 修复 OpenClaw 版本检测在某些安装方式下显示 unknown 的问题
