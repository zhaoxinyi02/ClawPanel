## v5.1.4 - Agents 页面官方工作区能力补齐

### 重点改进
- **feat**: 为单 Agent 新增 `核心文件 (Core Files)` 工作流
  - 新增后端接口：
    - `GET /api/openclaw/agents/:id/core-files`
    - `PUT /api/openclaw/agents/:id/core-files`
  - 仅允许访问核心文档白名单：
    - `AGENTS.md`
    - `SOUL.md`
    - `TOOLS.md`
    - `IDENTITY.md`
    - `USER.md`
    - `HEARTBEAT.md`
    - `BOOTSTRAP.md`
    - `MEMORY.md`
  - `Agents` 页面可直接查看并保存当前 Agent 工作区里的核心文档
  - review follow-up：
    - 只允许访问受管工作区目录下的核心文件
    - 拒绝工作区叶子、工作区祖先路径、受管根目录本身的 symlink 场景
    - Unix 平台使用目录 fd + `openat` / `renameat` 做安全读写
    - Windows 当前返回 unsupported-platform，避免继续暴露弱化版实现
    - 读取超过 512 KiB 的核心文件会截断，保存超过 512 KiB 的内容会被拒绝

- **feat**: `Agent 目录` 补齐官方式运行态快照
  - 新增 `技能与上下文 (Skills · Channels · Cron)` 页签
  - 展示当前 Agent 视角的：
    - skills 快照
    - channels 快照
    - cron jobs 快照
  - 新增 `最近会话 (Recent Sessions)`，帮助从单 Agent 角度快速巡检活跃上下文

- **feat**: 强化 `工具与权限 (Tools & Access)` 摘要
  - 展示 `profile / allow / deny / session visibility / delegation / sandbox` 等高频信息

### 后续修正：调试体验与启动保护
- **improve**: `OpenClawRequired` 支持关闭阻断提示，并切换为顶部非阻断 banner
  - 新增右上角关闭按钮
  - 新增“关闭提示，继续查看页面”与“重新显示提示”
  - 关闭状态按页面保存在本地 `sessionStorage`，方便前端调试页面结构且不会全局放开所有 OpenClaw 页面
- **improve**: `Agents` 顶部说明改成用户导向 copy
  - 使用 `使用方式 (How it works)` / `先管理单个智能体，再检查路由命中`
  - 明确区分 `Agent 目录` 与 `路由工作台` 的职责
- **fix**: ClawPanel 启动时若本机已存在真实 OpenClaw 控制台，将跳过 auto-start，避免重复拉起外部 OpenClaw
- **fix**: `process.Status` 新增 `managedExternally` 运行态，让状态接口与网关重启逻辑能识别外部管理中的 OpenClaw
- **fix**: `process.Status` 同时补充 `daemonized` 运行态，让快捷操作能把 daemon fork 模式引导到更安全的“网关重启”路径
- **fix**: 启动去重探测不再只看 loopback 端口占用，而是要求探测到真实 OpenClaw 控制台 HTTP 标识，降低误判风险
- **fix**: `gateway.bind=loopback/auto/custom-loopback` 的端口占用预检改为按绑定模式选目标地址，避免把其他网卡 / VPN / tailnet 地址上的监听误判成 loopback 端口冲突
- **fix**: `gateway.bind=custom` 会保留精确主机语义；`127.0.0.1` 与 `::1` 不再在启动预检里互相混用，避免把另一条 loopback 地址上的监听误算为当前自定义绑定冲突

### 后续修正：Windows CI `go vet` 收敛
- **fix**: 修复 `internal/monitor/launch_windows.go` 中 Windows 专属的 `unsafe.Pointer` 误用
  - `mergeEnvBlock()` 改为 typed pointer + `unsafe.Add(...)` 解析 UTF-16 环境块
  - `getSession1Username()` 改为 typed pointer + `unsafe.Slice(...)` 读取 `WTSQuerySessionInformationW` 返回缓冲区
- **fix**: `CreateProcessAsUserW` 使用的合并环境块现在会通过 `runtime.KeepAlive(...)` 保持到 syscall 返回之后
  - 避免把 Go slice 地址转成 `uintptr` 后在 GC 下出现悬空环境块风险
- **fix**: 当 `CreateEnvironmentBlock` 失败时，NapCat 启动 fallback 改为“当前进程环境基线 + `NAPCAT_*` overrides”
  - 避免 fallback 路径只带少量覆盖变量，导致缺失 `SystemRoot / PATH / TEMP` 等基础环境
