## v5.0.16 - OpenClaw Multi-Agent 对接

### 重点功能
- **feat**: 新增多智能体后端 API
  - `GET/POST/PUT/DELETE /api/openclaw/agents`
  - `GET/PUT /api/openclaw/bindings`
  - `POST /api/openclaw/route/preview`
- **feat**: 新增前端 `Agents` 页面（可通过 `VITE_FEATURE_AGENTS=false` 隐藏）
  - Agent 列表、新建/编辑/删除
  - bindings 规则顺序编辑与保存
  - 路由预览（meta → 命中 agent + trace）
- **feat**: 会话管理支持 `agent=all` 聚合查询并返回 `agentId`
- **feat**: 定时任务新建支持选择 `sessionTarget`，后端保存时校验目标 Agent 有效

### 配置保真与兼容性
- **fix**: `/api/openclaw/config` 读写不再删除 `tools` / `session` / `cron.jobs`
- **feat**: `openclaw.json` 读写支持 JSON5（注释、尾逗号）
- **feat**: 写配置前自动备份 `backups/pre-edit-*.json`，自动保留最近 10 份
- **feat**: `patchModelsJSON` 从单 `main` 改为遍历全部 Agent（含磁盘目录兜底）
- **feat**: 新增 `LEGACY_SINGLE_AGENT=true` 回滚开关，可快速退回单 Agent 兼容行为
- **feat**: bindings 兼容 `agentId/agent` 双字段，并支持顶层 `bindings` 与历史 `agents.bindings` 结构

### 测试与稳定性
- **test**: 新增后端测试覆盖
  - 配置保真保存（关键字段不丢失）
  - JSON5 解析与写前备份
  - `agent=all` 会话聚合
  - bindings 路由顺序命中
  - cron `sessionTarget` 非法值校验
- **fix**: 修复 `internal/updater/server.go` 中日志格式调用，消除 `go test`/vet 构建问题
