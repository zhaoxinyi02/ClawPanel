# 安全策略

感谢你帮助 ClawPanel 改进安全性。

## 反馈范围

欢迎反馈与以下内容相关的安全问题：

- 认证与授权缺陷
- 敏感信息泄露
- 配置写入或文件访问越权
- 更新链路与下载校验问题
- WebSocket / API 暴露导致的安全风险
- 安装脚本或服务脚本中的高风险行为

## 请不要公开提交漏洞细节

如果你发现疑似安全漏洞，请不要直接通过公开 Issue 披露完整利用细节。

建议通过以下方式联系：

- GitHub 私信仓库维护者
- 或先在仓库中创建不含利用细节的说明，再引导到私下沟通

如果你不确定是否属于安全问题，也可以先用 Discussions 简要说明现象，但请不要公开敏感 payload、密钥、令牌或完整利用步骤。

## 反馈时建议提供的信息

- 受影响版本
- 部署环境
- 漏洞类型与影响范围
- 最小复现步骤
- 是否需要登录 / 特定权限
- 可能的修复建议

## 处理原则

- 维护者会优先确认问题是否可复现
- 确认存在风险后，会尽快评估影响范围与修复优先级
- 在修复发布前，请尽量避免公开扩散漏洞细节

## 不在支持范围内的内容

以下内容通常不视为安全漏洞：

- 纯功能缺陷或 UI 表现问题
- 用户自行暴露密钥、Token、配置文件导致的泄露
- 第三方通道、插件或外部依赖本身已有的上游问题

感谢你的理解与支持。
